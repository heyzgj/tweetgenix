# TweetGenix - 产品需求文档 (PRD)

## 1. 产品概述

**产品名称**：TweetGenix

**产品类型**：Web 应用

**核心价值**：为用户提供 AI 驱动的推文生成工具，提升 Twitter 账号的互动率和曝光量

**目标用户**：
- 主要：个人创业者、投资人、加密货币 KOL
- 次要：内容创作者、社交媒体经理、普通 Twitter 用户

**成功衡量标准**：
- 生成的推文能显著提升用户的 Twitter 互动率和浏览量
- 用户觉得生成的内容自然、有价值，避免 AI 痕迹
- 用户界面简单直观，技术小白可以轻松使用

## 2. 用户流程

### 当前实现的流程

1. **访问应用**
   - 用户进入应用首页
   - 无需登录或注册即可使用核心功能

2. **输入生成条件**
   - 输入生成提示（如"分享一个创业小技巧"）
   - 粘贴参考推文（如 @elonmusk 的推文作为风格参考）
   - 选择推文字数范围（短：50-100字符、中：100-200字符、长：200-280字符）

3. **生成推文**
   - 点击"生成"按钮
   - 系统使用 Gemini 2.0 Flash 处理请求
   - 显示生成中的加载状态
   - 在 5 秒内返回生成结果

4. **查看和使用**
   - 查看生成的推文，显示在 Twitter 风格的卡片中
   - 可以直接编辑生成的文本
   - 点击复制按钮将内容复制到剪贴板
   - 生成的推文自动保存到历史记录

5. **历史记录**
   - 查看以往生成的所有推文
   - 按月份和日期分组展示
   - 可以搜索历史内容
   - 支持无限滚动加载更多历史记录

## 3. 功能规格

### 3.1 已实现功能

#### 推文生成引擎
- 使用 Gemini 2.0 Flash API
- 支持三种长度选项：短、中、长
- 使用参考推文作为风格基础
- 默认使用 casual 风格，适合创业者语境

#### 用户界面
- 简洁的输入表单
- Twitter 风格的结果展示
- 模拟互动数据展示（点赞、转发、评论）
- 复制到剪贴板功能
- 实时的加载状态和反馈

#### 历史记录
- 自动保存所有生成的推文
- 按月份和日期分组展示
- 实时搜索过滤功能
- 分页加载更多记录

### 3.2 移除的功能
- **"Saved"选项卡**：原计划中的分类保存功能已移除，简化了用户界面和交互流程
- 所有推文现在统一显示在历史记录中，不再区分"全部"和"已保存"类别

### 3.3 延后功能（未在当前版本实现）
- 用户账户系统
- 多条推文批量生成
- 高级编辑功能（添加图片、链接等）
- 直接发布到 Twitter 功能
- 推文效果反馈机制

## 4. 技术规格

### 4.1 技术栈
- **前端+后端**：Next.js (v15.x)，使用 App Router 和 Server Components
- **UI 框架**：基于 Tailwind CSS v4 的 shadcn/ui
- **AI 模型**：Gemini 2.0 Flash（通过 Google AI SDK）
- **数据库**：Supabase
- **部署环境**：Vercel

### 4.2 API 集成
- **Gemini API**：
  - 用于生成推文内容
  - 限制：免费层每天 100 次请求，每分钟 10 次请求

### 4.3 数据结构
- **推文数据模型**：
  ```typescript
  interface Tweet {
    id: string;           // 唯一标识
    content: string;      // 推文内容
    created_at: string;   // 创建时间
  }
  ```

### 4.4 性能要求
- 推文生成时间：≤ 5 秒
- 页面加载时间：≤ 2 秒
- 应用大小：≤ 5 MB

## 5. 设计规格

### 5.1 视觉设计
- 界面风格：极简，Twitter/X 平台风格
- 色彩方案：
  - 主色：Twitter 蓝 (#1d9bf0)
  - 背景色：白色 (#ffffff)
  - 文本色：深灰 (#0f1419)
  - 次要文本：中灰 (#536471)
- 字体：'Chirp'（Twitter 字体）或系统无衬线字体

### 5.2 响应式设计
- 桌面端（≥ 1024px）：全功能布局
- 平板端（768px-1023px）：适应性布局
- 移动端（< 768px）：垂直堆叠布局

## 6. 限制与约束

### 6.1 技术限制
- Gemini API 调用限制（免费层每天 100 次）
- Vercel 和 Supabase 免费层的带宽和存储限制
- 不支持跨浏览器复制（如 Safari 的复制限制）

### 6.2 业务限制
- 无认证系统，无法保存用户特定数据
- 无法直接发布到 Twitter，依赖用户手动复制粘贴
- 无法获取发布后的推文效果数据

## 8. 已知问题与解决方案

### 8.1 当前问题
- **免费额度限制**：
  - 问题：每日 100 次 API 请求可能不足
  - 解决方案：实施请求计数和用户提示，显示剩余次数

- **无直接发布**：
  - 问题：用户需要手动复制粘贴
  - 解决方案：优化复制体验，提供明确的操作成功反馈

- **生成内容质量**：
  - 问题：生成质量可能不稳定
  - 解决方案：持续调整提示工程，优化 AI 输入模板

### 8.2 监控指标
- 生成请求成功率
- 平均生成时间
- 复制操作使用率
- 历史记录查看频率
- 用户会话时长
